## Summary

Anthropic 发布的创业者指南，系统描述了 AI 原生创业公司在 Idea → MVP → Launch → Scale 四个阶段的挑战、工具应用和退出条件。Claude、Claude Code、Claude Cowork 三款产品各司其职：Claude 负责研究与战略、Claude Code 负责产品构建、Claude Cowork 负责运营自动化。

核心主张：AI 时代创业者的工作没有变（找真实问题、构建解决方案、规模化），变的是路径——AI 把季度压缩成周。

## Key Themes

### Chapter 1: Startup lifecycle rebooted
- 传统创业路径被压缩：验证从月→午，原型从团队→独自构建，运营从人工→AI接管
- 关键转变：瓶颈不再是"能构建什么"，而是"选择构建什么"

### Chapter 2: 创始人角色演变
- 创始人从"构建者"变为"编排者"（orchestrator）
- 三大 AI 能力：研究与验证 / 智能体编码 / 工作流自动化
- Claude Cowork 作为新产品：AI 智能体团队，管理日历、收件箱、项目、运营层

### Chapter 3: Idea Stage
- 四大挑战：过早执行、假设性 PMF、竞争盲点、创始人偏见
- AI 帮助：快速研究市场、模拟反驳、分析竞品
- 退出条件：用户访谈 + 数据 + 分析支撑的 PMF 假设

### Chapter 4: MVP Stage
- 四大挑战：
  1. **智能体技术债**（Agentic Technical Debt）：AI 生成快但脆弱，需 CLAUDE.md + 测试 + 文档
  2. 虚假 PMF：A/B 测试早期假设
  3. 功能蔓延：明确 MVP 边界
  4. 安全：从第一天就扫描
- CLAUDE.md 作为架构上下文：产品目标、技术决策、边界、测试覆盖要求
- 退出条件：真实用户的可重复使用模式

### Chapter 5: Launch Stage
- 挑战：技术债清理、创始人瓶颈、安全合规
- Claude Cowork 管理运营层：Sprint 安排、bug 路由、指标汇编
- 运营 OS：Sprint 节奏 + 最小规格模板 + Bug 分类决策树 + 周报
- 退出条件：系统在创始人不在场时仍能运行

### Chapter 6: Scale Stage
- 创始人角色：从构建者 → 对外执行官
- 四大挑战：
  1. 委托运营层（心理转变 + 结构转变）
  2. 建立 GTM 功能（市场细分、消息架构、分析师关系）
  3. 扩展技术运营至企业级（SLA、监控、事件响应）
  4. 扩展组织职能（招聘、财务、法务）
- 关键武器：
  - **数据飞轮**：行为信号 → 产品改进 → 更多使用 → 更多反馈
  - **工作流锁定**：深度集成让切换成本变为运营项目
  - **领域专业知识 → AI 上下文**：把行业特有 edge case 编入产品

### Chapter 7: Same job, new rules
- AI 时代创业者的工作未变：找问题、构建方案、规模化
- 变化：路径压缩，瓶颈从"能力"转为"选择"
- 核心结论："The bottlenecks are no longer what you can build, but what you choose to build."
