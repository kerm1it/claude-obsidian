# RuView — WiFi DensePose: Spatial Intelligence via WiFi Signals

GitHub: ruvnet/RuView | Stars: 59,973 | Forks: 7,814 | Language: Rust | License: MIT

## Description

RuView turns commodity WiFi signals into real-time spatial intelligence, vital sign monitoring, and presence detection — all without a single pixel of video. It uses Channel State Information (CSI) from low-cost ESP32-S3 sensors ($9) to detect people through walls, measure breathing and heart rate, estimate human poses (17 COCO keypoints), and perform activity recognition.

## Core Capabilities

- **Presence and occupancy** — detect people through walls, count them, track entries/exits
- **Vital signs** — breathing rate (6-30 BPM) and heart rate (40-120 BPM), contactless
- **Pose estimation** — 17 COCO keypoints via WiFlow architecture (171K emb/s on M4 Pro)
- **Activity recognition** — walking, sitting, gestures, falls from temporal CSI patterns
- **Sleep quality** — overnight monitoring with sleep stage classification and apnea screening
- **Through-wall sensing** — up to 5m depth via Fresnel zone geometry + multipath modeling
- **3D point cloud (optional)** — camera depth (MiDaS) + WiFi CSI + mmWave radar fusion

## Key Architecture Components

### Hardware
- **ESP32-S3** sensor nodes ($9 each); ESP32-C3 and original ESP32 NOT supported
- **Cognitum Seed** ($140 total BOM with ESP32) — persistent vector store, kNN search, Ed25519 witness chain, MCP proxy
- **Research NICs**: Intel 5300 / Atheros AR9580 for full CSI with 3x3 MIMO

### Software
- **RuVector** — AI backbone for signal processing (shared with Cognitum Seed)
- **rvCSI** — edge RF sensing runtime (Rust-first, TypeScript-accessible, now its own repo: ruvnet/rvcsi)
- **65 Edge Modules** — no_std Rust compiled to WASM32, run on ESP32 via WASM3
- **WiFlow architecture** — 17-keypoint pose estimation trained without cameras

### Self-Learning WiFi AI (ADR-024)
- ~55KB model (28K transformer + 25K embedding head) fits on ESP32
- Self-supervised contrastive learning — no camera labels needed
- MicroLoRA adapters (1,792 params/room) for per-environment fine-tuning
- EWC++ regularization for catastrophic forgetting prevention
- 84 seconds to train camera-free on M4 Pro; adapts to new room in 30 seconds

### Multi-frequency Mesh
- 6 WiFi channel hopping
- Neighbor access points used as free radar illuminators (3x sensing bandwidth)
- Ed25519 cryptographic witness chain for attestation
