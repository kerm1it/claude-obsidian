# AI 体系学习路线

这条路线从 [[domains/AI知识体系]] 派生，目标不是追热点，而是形成一个稳定心智模型：新知识出现时，能判断它属于哪一层、连接哪些上下游、是否值得深学。

它和 [[learning/AI学习计划-2026]] 是一套东西的两面：本页负责“为什么按这个顺序学”和“概念属于哪一层”；[[learning/AI学习计划-2026]] 负责“今天/本周具体看什么资源、做什么练习、怎么记录进度”。

## 学习原则

- 先学链路，再学单点。
- 先理解“为什么需要这一层”，再理解工具或论文细节。
- 每学一个概念，都补一行：主归属、上游输入、下游输出、评估方式。
- 遇到新名词不要立刻扩体系，先挂到 8 层之一。

## 阶段 1：模型能力从哪里来

覆盖层级：1 基础理论层、2 模型构建层、3 推理与能力层。

学习顺序：
1. Transformer：注意力机制为什么取代 RNN/CNN 成为通用序列架构。
2. Scaling Laws：为什么规模、数据和算力能系统性提升能力。
3. 预训练：为什么 next-token / masked prediction 能形成通用表示。
4. 指令微调与 RLHF：为什么基础模型还需要被对齐到用户意图。
5. 微调、PEFT、LoRA：什么时候把任务知识写进权重。
6. 蒸馏：如何把大模型或 ensemble 的能力迁移到更便宜的模型。
7. 推理时计算：为什么运行时预算也能换能力，先读 [[concepts/TestTimeCompute]] 和 [[Research: 推理时计算与 reasoning]]。
8. Reasoning eval / verifier：理解 outcome verifier、process verifier、trace grader 和 reward model 如何控制推理质量，先读 [[concepts/ReasoningEvalVerifier边界]]。
9. 多模态 / 世界模型：区分多信号理解、生成、动态模拟和行动后果，先读 [[concepts/多模态与世界模型边界]]。
10. Multimodal / simulation eval：理解静态多模态、视频、生成安全、world-model 和 embodied eval，先读 [[concepts/MultimodalEvalSimulationEval边界]]。
11. Serving / 成本 / 延迟：理解 prefill/decode、KV cache、batching、prompt caching 和 cost/success，先读 [[concepts/Serving成本延迟边界]]。
12. Model routing / mixture：理解模型选择、cascade、learned router、MoE、MoA 和 compound system allocation 的边界，先读 [[concepts/ModelRoutingMixture边界]]。

实践任务：
- 写一张“改 prompt / RAG / fine-tune / distill”的选择表。
- 对一个具体场景判断：该补上下文、改工具、微调，还是做 eval。
- 参考 [[concepts/微调RAG上下文工程选择框架]]，为自己的一个真实任务做一次路径选择。

## 阶段 2：模型如何接入真实任务

覆盖层级：4 上下文与知识层、5 Agent 系统层、6 评估与可靠性层。

学习顺序：
1. Prompt engineering：单次交互的指令组织。
2. Context engineering：多轮任务中管理全部 token。
3. RAG：把外部知识变成可追溯上下文。
4. Memory / skills：把可复用经验外化到模型窗口外，先读 [[concepts/RAG与Memory边界]]、[[concepts/Skill与ToolMemory边界]] 和 [[concepts/SkillLibraryRouting生命周期]]。
5. ReAct / tool use：把推理和行动交错起来。
6. Agent harness：把模型、工具、文件、状态、沙盒和 trace 组织成运行时，先读 [[concepts/AgentHarness与TraceEval]]。
7. Agent identity / delegation：理解 agent 以谁的身份行动、谁授权、权限多大、多久过期、如何审计，先读 [[concepts/AgentIdentityDelegation边界]]。
8. Tool risk / permissioning：理解工具动作如何按风险层进入权限、沙箱、审批、审计和回滚，先读 [[concepts/ToolRiskPermissioning边界]]。
9. Reasoning / trace eval：区分最终答案评分、过程评分、trace grading、CoT monitor 和 reward model。
10. Multimodal / embodied eval：当 agent 进入视觉、音频、3D 或机器人环境时，评估最终状态、物理一致性、行动安全和泛化。
11. Data quality / label quality：理解反馈、label、trace、RAG 文档和训练候选如何进入质量门，先读 [[concepts/DataQualityLabelQuality边界]]。
12. Synthetic data governance：理解合成样本如何按用途进入训练、eval、RAG、memory、skill 或 self-improvement，先读 [[concepts/SyntheticDataGovernance边界]]。
13. Dataset lineage / provenance graph：理解样本、trace、RAG、memory、skill、eval、training 和模型版本之间如何形成可查询证据链，先读 [[concepts/DatasetLineageProvenanceGraph边界]]。
14. Human feedback / annotation ops：理解人类示范、偏好、纠错、gold label 和 expert review 如何经过 rubric、校准、QA、仲裁和 lineage 变成证据，先读 [[concepts/HumanFeedbackAnnotationOps边界]]。
15. LLM-as-judge / AI feedback calibration：理解模型裁判、AI 标注、RLAIF teacher 和 multi-judge panel 如何用 human gold、bias suite、生产结果校准，先读 [[concepts/LLMJudgeAIFeedbackCalibration边界]]。
16. Judge drift / grader observability：理解 judge model、prompt、rubric、human anchor、score distribution 和 production correlation 如何随时间被监控，先读 [[concepts/JudgeDriftGraderObservability边界]]。
17. Evaluation dataset lifecycle / benchmark governance：理解 eval set、private hold-out、public benchmark、regression suite、red-team set 如何创建、分流、版本、泄漏防护、刷新和退役，先读 [[concepts/EvaluationDatasetLifecycleBenchmarkGovernance边界]]。
18. Eval result interpretation / decision threshold：理解 score、baseline、uncertainty、guardrail、slice、SLO 和风险阈值如何变成 pass/fail/gray-zone/canary/rollback，先读 [[concepts/EvalResultInterpretationDecisionThreshold边界]]。
19. Eval uncertainty communication / release card：理解不确定性、灰区、局限、hard stop、owner 和 post-release monitor 如何写成决策者能审查的证据卡，先读 [[concepts/EvalUncertaintyCommunicationReleaseCard边界]]。
20. Offline / online eval correlation：理解离线 eval、release card 和线上 outcome 是否同向，以及如何决定指标 trust tier、refresh 或 retire，先读 [[concepts/OfflineOnlineEvalCorrelation边界]]。
21. Post-release eval drift：理解发布后如何判断 release card、threshold、offline/online correlation、judge 和 safety evidence 是否变 stale，先读 [[concepts/PostReleaseEvalDrift边界]]。
22. Eval portfolio / metric hierarchy：理解 benchmark、hold-out、regression、red-team、judge/human、online 和 SLO 如何组成 hard stop、guardrail、primary、slice、diagnostic 等分层证据，先读 [[concepts/EvalPortfolioMetricHierarchy边界]]。
23. Metric conflict resolution：理解 hard stop、guardrail、primary、slice、online、SLO 和 business metric 冲突时如何排序、升级和行动，先读 [[concepts/MetricConflictResolution边界]]。
24. Evidence freshness / stale proof：理解 eval、release card、safety case、monitor、judge 和 benchmark 证据何时 fresh、stale、expired、superseded、invalidated 或 unknown，先读 [[concepts/EvidenceFreshnessStaleProof边界]]。
25. Evidence attestation / signed eval artifact：理解 eval result、model、prompt、judge、dataset、container 和 release evidence 如何用 digest、signature、producer、trust root 和 verifier policy 变成可验证证据，先读 [[concepts/EvidenceAttestationSignedEvalArtifact边界]]。
26. Verifier policy / trust root rotation：理解 signed evidence 如何按当前 policy、identity、issuer、predicate、trust root、TUF metadata 和 rotation/revocation 状态被接受或拒绝，先读 [[concepts/VerifierPolicyTrustRootRotation边界]]。
27. Cross-verifier policy drift / conformance test：理解 CI、release gate、admission controller、本地 verifier 和第三方 verifier 如何用同一 policy/root/corpus 产生一致验收结果，先读 [[concepts/CrossVerifierPolicyDriftConformanceTest边界]]。
28. Evidence minimization / privacy-preserving audit packet：理解 audit packet 如何用最小充分证据证明 claim，并避免长期保留或披露敏感 trace、PII、secret 和 hold-out，先读 [[concepts/EvidenceMinimizationPrivacyPreservingAuditPacket边界]]。
29. Release-gate debt / gate bypass detection：理解真实生产变更如何证明经过 release gate、protected environment、approval、attestation 或 structured exception，先读 [[concepts/ReleaseGateDebtBypassDetection边界]]。
30. Dashboard calibration / governance metric quality：理解治理看板指标如何定义、校准、分层、触发行动和退役，先读 [[concepts/DashboardCalibrationGovernanceMetricQuality边界]]。
31. Production contract / compatibility test：理解产品行为、schema、工具、安全、数据处理和 SLO 如何变成模型/provider 更新前的可测试契约，先读 [[concepts/ProductionContractCompatibilityTest边界]]。
32. Reward hacking / eval overfitting：理解 eval、reward、benchmark、KPI 被优化压力污染的方式，先读 [[concepts/RewardHackingEvalOverfitting边界]]。
33. Agent eval：从输出评分升级为状态、trace、工具调用、权限决策和成本评估。

实践任务：
- 为一个研究 agent 写出上下文清单：系统提示、工具、记忆、来源、scratchpad。
- 设计 10 个真实失败案例，并把它们变成 eval。

## 阶段 3：从能力到产品和组织

覆盖层级：7 产品与组织层、8 自我改进与前沿层。

学习顺序：
1. Product / organization layer：先读 [[concepts/ProductOrganizationLayer边界]]，理解可靠能力如何变成产品表面、工作流、组织角色和反馈数据。
2. 模型即产品：模型能力如何由用户反馈和 eval 驱动迭代。
3. 软件工厂：spec + tests + agent loop 如何重写工程流程。
4. 公司大脑：组织知识如何变成 agent 可查询上下文。
5. 数据飞轮 / 工作流锁定：理解第 7 层如何把真实使用反馈和深度集成变成复利，先读 [[concepts/DataFlywheelFeedbackLoop边界]]。
6. Product feedback closure / actionability：理解 feedback 如何从日志和看板变成有 owner、artifact、验证、关闭证据和 reopen 条件的行动，先读 [[concepts/ProductFeedbackClosureActionability边界]]。
7. Data rights / privacy / consent：理解哪些数据可以被收集、记忆、评估、训练或发给供应商，先读 [[concepts/DataRightsPrivacyConsent边界]]。
8. Data deletion / unlearning impact：理解删除请求如何穿透 logs、RAG、memory、eval、training、model weights 和 providers，先读 [[concepts/DataDeletionUnlearningImpact边界]]。
9. AI safety ops / governance：理解风险证据如何变成 owner、release gate、incident response 和 audit trail，先读 [[concepts/AISafetyOpsGovernance边界]]。
10. Monitor ownership / escalation ladder：理解第 6 层 monitor、eval breach、stale evidence 和 user report 如何接到 owner、SLA、incident、release gate 或 risk elevation，先读 [[concepts/MonitorOwnershipEscalationLadder边界]]。
11. Monitor tabletop / runbook drill：理解 monitor owner、on-call、incident command、release gate、risk owner 和 provider POC 如何通过场景、injects、drill、AAR/IP 被验证，先读 [[concepts/MonitorTabletopRunbookDrill边界]]。
12. Override governance / residual risk acceptance：理解例外如何在不绕过质量门的前提下，绑定 hard-stop check、risk owner、expiry、monitor、rollback 和复审，先读 [[concepts/OverrideGovernanceResidualRiskAcceptance边界]]。
13. Override debt / exception analytics dashboard：理解例外率、过期例外、重复续期、重复 owner、risk-weighted exposure 和 incident conversion 如何变成组织治理反馈，先读 [[concepts/OverrideDebtExceptionAnalyticsDashboard边界]]。
14. Dashboard calibration / governance metric quality：理解治理看板指标是否可信、可比、可行动，先读 [[concepts/DashboardCalibrationGovernanceMetricQuality边界]]。
15. Intolerable risk threshold / stop rule：理解哪些风险不能被平均收益或总体分数抵消，以及如何触发 no-go、pause、restrict、rollback 或 external review，先读 [[concepts/IntolerableRiskThresholdStopRule边界]]。
16. Safety case / assurance case：理解高风险系统如何把 eval、red-team、release card、monitoring 和 residual risk 组织成可审查发布论证，先读 [[concepts/SafetyCaseAssuranceCase边界]]。
17. Model release / rollback gate：理解模型、prompt、router、skill、tool、provider 变更如何进入 eval gate、灰度、发布、回滚和迁移，先读 [[concepts/ModelReleaseRollbackGate边界]]。
18. Release-gate debt / gate bypass detection：理解真实生产变更如何发现 admin bypass、shadow approval、direct-to-prod、stale exception 和 gate config drift，先读 [[concepts/ReleaseGateDebtBypassDetection边界]]。
19. Evidence retention / audit packet lifecycle：理解 release、eval、safety case、incident 和 override 证据如何长期可检索、可重验、可 legal hold、可销毁，先读 [[concepts/EvidenceRetentionAuditPacketLifecycle边界]]。
20. Multi-provider governance：理解多模型、多云和多工具供应商如何先经过数据、地域、retention、training use、SLA 和风险过滤，再进入 routing，先读 [[concepts/MultiProviderGovernance边界]]。
21. AI incident response / postmortem：理解监控/eval/用户报告如何进入止血、取证、沟通、复盘和 regression eval，先读 [[concepts/AIIncidentResponsePostmortem边界]]。
