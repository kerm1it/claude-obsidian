# Headroom — The context compression layer for AI agents

Tejas Chopra（@chopratejas）开源的 AI Agent 上下文压缩层。14.1K ⭐，Python+TypeScript，Apache 2.0。

## 核心主张

在工具输出、日志、RAG 结果、文件和对话历史到达 LLM 之前压缩——**60-95% token 节省，答案不变**。本地运行，数据不离开。可逆压缩（CCR）——原文不删除，LLM 按需取回。

## 六种接入方式

| 方式 | 描述 |
|------|------|
| Library | `compress(messages)` in Python/TypeScript |
| Proxy | `headroom proxy --port 8787`，零代码 |
| Agent wrap | `headroom wrap claude\|codex\|cursor` |
| MCP server | `headroom_compress` / `headroom_retrieve` / `headroom_stats` |
| Cross-agent memory | 共享存储，跨 Claude/Codex/Gemini，自动去重 |
| headroom learn | 挖掘失败 session → 写回 CLAUDE.md/AGENTS.md |

## 六种压缩算法

- **SmartCrusher**：通用 JSON 压缩（数组、嵌套对象、混合类型）
- **CodeCompressor**：AST 感知（Python/JS/Go/Rust/Java/C++）
- **Kompress-base**：HuggingFace 模型，基于 agentic traces 训练
- **CacheAligner**：稳定前缀 → Anthropic/OpenAI KV cache 命中
- **IntelligentContext**：基于分数的上下文拟合+学习重要性
- **CCR**：可逆压缩，原文本地存储，LLM 按需 `headroom_retrieve`

## 效果数据

| 场景 | 压缩前 | 压缩后 | 节省 |
|------|--------|--------|------|
| Code search (100 results) | 17,765 | 1,408 | **92%** |
| SRE incident debugging | 65,694 | 5,118 | **92%** |
| GitHub issue triage | 54,174 | 14,761 | 73% |
| Codebase exploration | 78,502 | 41,254 | 47% |

准确性：GSM8K ±0.000 / TruthfulQA +0.030 / SQuAD v2 97% / BFCL tools 97%。

## Agent 兼容

Claude Code (`--memory` `--code-graph`) / Codex（共享记忆）/ Cursor / Aider / Copilot CLI / OpenClaw（ContextEngine 插件）。任何 OpenAI 兼容客户端通过 proxy。

## 管线生命周期

11 阶段请求管线：Setup→Pre-Start→Post-Start→Input Received→Cached→Routed→Compressed→Remembered→Pre-Send→Post-Send→Response Received。支持扩展和 hooks。

## headroom learn

插件化失败挖掘：分析失败 session → 提取修正 → 写回 CLAUDE.md / AGENTS.md / GEMINI.md。→ 与 [[concepts/DynamicWorkflows]] Memory and rule adherence 模式收敛。
