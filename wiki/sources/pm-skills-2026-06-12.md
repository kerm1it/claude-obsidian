# PM Skills Marketplace

**来源**：phuryn  
**GitHub**：https://github.com/phuryn/pm-skills  
**许可**：MIT | ⭐ 16K | v2.0.0（2026-06-05）| 60 commits

---

## 一句话定位

> "Generic AI gives you text. PM Skills Marketplace gives you structure."

面向产品管理的 Agent Skills 市场——68 个 skills、42 个 commands、9 个可安装 plugins，覆盖从发现到策略、执行、发布和增长的完整 PM 工作流。

---

## 架构：Skills → Commands → Plugins 三层

```
Skills（原子技能，自动加载）
  → Commands（用户触发的斜杠命令，串联多个 skills）
    → Plugins（可安装的领域包，包含 skills + commands）
```

| 层 | 数量 | 触发方式 | 粒度 |
|----|------|---------|------|
| Skill | 68 | 上下文匹配自动加载 | 单一框架/分析工具 |
| Command | 42 | 用户输入 `/command` | 端到端工作流（串联多个 skills） |
| Plugin | 9 | 用户安装 | 领域完整技能包 |

Commands 完成后会建议相关下一步命令，匹配自然 PM 工作流。

## 关键设计决策

1. **PM 框架编码**：每个 skill 将 Teresa Torres、Marty Cagan、Alberto Savoia 等人的框架编码为可执行的引导工作流
2. **跨平台兼容**：Skills 使用通用 Agent Skills 格式（SKILL.md），commands 是 Claude 专用，skills 可被任何兼容平台使用
3. **配套 PM Brain**：与 [[entities/PMBrain]]（纯 Markdown 第二大脑）集成，无向量库或云依赖
4. **AI Shipping 插件**：解决"AI 写代码快但不留意图"的痛点——文档化系统并审计文档与代码的差距

---

## 关键洞察

1. **领域垂直化是 Agent Skills 的下一阶段**：从通用科学技能（Scientific Agent Skills）到垂直领域（PM），每个垂直有自己的框架、工作流和用户画像
2. **三层架构解决了"安装什么"的问题**：atomic skill → 可组合 command → 可发现 plugin，降低了用户的认知负载
3. **16K ⭐ 说明 PM 群体对结构化 AI 引导有强需求**：不是"让 AI 替你写 PRD"，而是"让 AI 用已建立框架引导你写 PRD"
4. **技能市场模式 vs 技能仓库模式**：pm-skills 是 marketplace（用户选装 plugin），Scientific Agent Skills 是 repository（全部安装）。前者更适合垂直领域
