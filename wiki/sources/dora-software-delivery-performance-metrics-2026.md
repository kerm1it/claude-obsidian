# DORA Software Delivery Performance Metrics

**来源**：DORA
**URL**：https://dora.dev/guides/dora-metrics/

## 摘要

DORA 将软件交付表现分成 throughput 和 instability，使用 change lead time、deployment frequency、failed deployment recovery time、change fail rate、deployment rework rate 等指标观察交付能力和变更稳定性。

## 关键贡献

- 软件交付指标既可作为 leading indicators，也可作为 lagging indicators。
- 指标分为 throughput 与 instability，避免只追速度而忽视失败和返工。
- DORA 指标随实践演化，从四指标扩展到五指标，说明指标口径需要持续校准。
- 对本 vault 的意义：AI governance dashboard 也应把速度、质量、风险、返工和恢复放在同一测量系统中，而不是单独优化一个分数。
