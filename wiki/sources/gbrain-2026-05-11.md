# GBrain — AI Agent 记忆与知识管理系统

> [!key-insight] 核心定位
> GBrain 是让 AI Agent 拥有持久记忆的操作系统。Garry Tan（YC 总裁）在自己的 OpenClaw 和 Hermes 部署中实际运行，管理 17,888 个页面、4,383 人、723 家公司，挂载 21 个自主 cron 任务。

**来源**：[[people/GarryTan]] · GitHub repo · 14.6K ⭐ · 1.9K forks · MIT · TypeScript 98%

---

## 是什么

[[entities/GBrain]] 是一个 AI Agent 记忆与知识管理系统——为"聪明但健忘"的 Agent 提供持久的、可搜索的上下文。它不是聊天记录，而是一个结构化知识图谱 + 混合检索 + 自动维护的完整系统。

## 核心能力

### 知识管理
- **混合检索**：向量嵌入 + 关键词匹配 + 互惠排名融合（[[concepts/混合检索RRF|RRF]]）
- **自编织知识图谱**：类型化关系（attended / works_at / invested_in / founded / advises）
- **[[concepts/CompiledTruth时间线模式|Compiled Truth + Timeline 模式]]**：所有页面拆分为编译事实区 + 时间线追加区
- 自动实体抽取 + 反向链接生成

### Agent 集成
- 34 个专用 skill，覆盖摄入、富化、脑操作、研究
- [[concepts/Minions任务队列|Minions 任务队列]]：确定性后台工作
- MCP 服务器支持（Claude、Cursor、Windsurf、远程 OAuth）
- 可恢复的子 agent 执行（崩溃恢复）

### 数据摄入
- 会议、邮件、推文、语音通话、文章、PDF
- Gmail、Calendar、X/Twitter、Twilio Voice 集成配方

## 技术架构

| 层次 | 选型 |
|------|------|
| 运行时 | Bun |
| 语言 | TypeScript (98%) |
| 存储（本地） | PGLite（嵌入式 Postgres） |
| 存储（生产） | Supabase Postgres |
| 向量引擎 | pgvector + HNSW |
| 嵌入模型 | OpenAI Embeddings API |

**搜索流水线**：意图分类 → 多查询扩展 → HNSW 向量 + tsvector 关键词 → RRF 融合去重 → 编译事实加权 + 反向链接排名
