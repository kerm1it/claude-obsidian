# RuView — WiFi DensePose：用 WiFi 信号实现空间智能

**来源**：[github.com/ruvnet/RuView](https://github.com/ruvnet/RuView)
**摄入日期**：2026-05-19
**Stars**：59,973 ⭐ | **Forks**：7,814 | **语言**：Rust | **License**：MIT

## 一句话总结

RuView 把普通 WiFi 路由器变成穿墙感知系统：无摄像头、无可穿戴设备，仅凭 ESP32-S3 传感器（$9）捕获 Channel State Information（CSI），即可实时检测人员存在、测量呼吸与心率、估计人体姿态（17 个 COCO 关键点）。

## 核心能力

| 能力 | 说明 | 指标 |
|------|------|------|
| 存在与占位 | 穿墙检测、计数、进出追踪 | 100% 准确率，0.012ms 延迟 |
| 生命体征 | 非接触呼吸（6-30 BPM）+ 心率（40-120 BPM） | — |
| 人体姿态 | WiFlow 架构，17 个 COCO 关键点 | 171K emb/s（M4 Pro）|
| 穿墙感知 | Fresnel 区域几何 + 多径建模 | 最深 5m |
| 睡眠监测 | 整夜睡眠分期分类 + 呼吸暂停筛查 | — |
| 3D 点云（可选）| 摄像头深度 + CSI + 毫米波融合 | 22ms，19K+ 点/帧 |

## 系统架构

### 硬件
- **ESP32-S3**（$9/节点）：CSI 流采集；ESP32-C3 和原版 ESP32 不支持（单核不足）
- **Cognitum Seed**（~$140 全套 BOM）：持久向量存储、kNN 搜索、Ed25519 见证链、MCP 代理 → [[entities/CognitumSeed]]
- **研究级网卡**：Intel 5300 / Atheros AR9580，3x3 MIMO 完整 CSI

### 软件栈
- **[[entities/RuVector]]**：AI 骨干，信号处理层（与 Cognitum Seed 共用）
- **rvCSI**：边缘 RF 感知运行时（Rust-first，TypeScript 可访问；已独立为 ruvnet/rvcsi 仓库）
- **65 个 Edge Modules**：no_std Rust 编译到 WASM32，通过 WASM3 在 ESP32 上运行 → [[concepts/WASM边缘模块]]
- **WiFlow 架构**：无摄像头训练的 17 关键点姿态估计 → [[concepts/WiFiDensePose]]

### 自监督 WiFi AI（ADR-024）
→ [[concepts/自监督WiFi学习]]
- 模型仅 ~55KB，可运行于 ESP32（520KB 可用）
- 对比学习，无需摄像头标签；单环境 MicroLoRA 适配器（1,792 参数），30 秒内适应新房间
- EWC++ 正则化防灾难性遗忘；M4 Pro 上 84 秒无摄像头训练

### 多频 Mesh
- 跨 6 个 WiFi 信道跳频
- 邻居 AP 作为免费雷达照射源（感知带宽 3×）
- Ed25519 密码学见证链认证每次测量

## Claude Code / Codex 插件

`plugins/ruview/`：9 个 skills，7 条 `/ruview-*` 指令，3 个 Agent。

```bash
/ruview-start      # 入门引导（Docker 演示 / 仓库构建 / 实体 ESP32）
/ruview-flash      # 构建 + 烧录 ESP32 固件
/ruview-provision  # WiFi 凭证、Sink IP、信道/MAC、Mesh 槽位
/ruview-app        # 运行感知应用（存在/生命体征/姿态/睡眠/MAT/点云）
/ruview-train      # 训练/评估/发布模型（含 GCloud GPU）
/ruview-advanced   # 多静态/层析/交叉视点/Mesh 安全
/ruview-verify     # 测试 + 确定性证明 + 见证包
```

Codex 镜像：`cp plugins/ruview/codex/prompts/*.md ~/.codex/prompts/`

## 学术来源

基于卡内基梅隆大学 *DensePose From WiFi* 研究。无摄像头基准 PCK@20 ≈ 2.5%（代理标签）；摄像头监督目标为 **35%+ PCK@20**（ADR-079，流水线已实现，评估阶段待完成）。
