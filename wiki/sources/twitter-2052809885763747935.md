# HTML 的非凡效能 —— Claude Code 最佳输出格式

## 核心主张

> 用 **HTML** 而非 Markdown 作为 [[concepts/HTML作为AI输出格式|Claude 生成产出物]]的首选格式。

## 五大理由

| 理由 | 说明 |
|------|------|
| **信息密度** | 支持表格、CSS、SVG、交互元素，Markdown 无法比拟 |
| **视觉清晰度** | 文档更易读易分享 |
| **分享便利** | 直接上传链接，无需附件 |
| **双向交互** | 滑块、表单、可编辑元素让读者可以与文档互动 |
| **数据摄入** | Claude Code 可通过文件系统、MCP、浏览器历史、git 日志获取更广上下文 |

## 适用场景

- 规格文档和实现规划
- 代码审查报告
- 设计原型和组件
- 研究综合报告
- 针对特定任务的定制编辑界面

## 权衡

HTML 生成比 Markdown 慢 **2–4 倍**，且 git diff 噪声更高，在协作开发环境中是明显缺点；但作者认为清晰度和交互性的提升值得。
