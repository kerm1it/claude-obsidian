# 代码库图谱

**来源**：[[entities/Hivemind]]（Activeloop，2026）  
**所属体系层**：第 4 层（上下文与知识）+ 第 5 层（Agent 系统）

---

## 是什么

代码库图谱是从 Agent 实际交互 traces 构建的**实时代码结构图**。它不是静态 AST 分析，而是 Agent 在真实会话中遍历的文件、符号、导入和依赖关系的沉淀。

---

## 与传统代码搜索的区别

| 维度 | 传统搜索（grep/向量） | 代码库图谱 |
|------|---------------------|----------|
| 数据源 | 代码文件内容 | Agent traces（实际访问路径） |
| 构建方式 | 静态索引 | Agent 交互增量构建 |
| 回答的问题 | "哪些文件提到 auth？" | "团队的 Agent 实际在哪处理 auth？" |
| 时效性 | 依赖定期重建索引 | 每次会话更新 |
| 噪声 | 返回所有匹配（包括未使用的代码） | 返回 Agent 真正碰过的 |

---

## 构建方式

```
Agent session → 每次 tool call（读文件、搜索符号、追踪导入）
  → 记录为图谱的节点（文件/符号）和边（读/导入/搜索）
  → 边权重按访问频率和成功率更新
  → 搜索时走图谱路径，而非全文匹配
```

---

## 价值

1. **降低搜索噪声**：Agent 不会浪费时间在看"提到 auth 但从没被修改过的 50 个文件"上
2. **团队知识沉淀**：图谱反映了团队的实际开发路径，新人 Agent 可以"顺着前人的脚印"走
3. **与共享记忆协同**：图谱是共享记忆的一种特殊形态——不存"内容"，存"结构关系"
