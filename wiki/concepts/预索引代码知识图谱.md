# 预索引代码知识图谱

**预索引代码知识图谱（Pre-indexed Code Knowledge Graph）** 是一种 AI 编码 Agent 基础设施模式：在 Agent 查询之前，预先用 AST 解析器（通常是 tree-sitter）分析代码库，将符号（函数、类、方法、类型、路由）构建为节点，将关系（调用、导入、继承、引用）构建为边，存入本地图数据库（通常 SQLite + FTS5），Agent 通过 MCP 工具直接查询精确上下文。

## 与传统方法对比

| | grep + Read | 预索引知识图谱 |
|---|---|---|
| 发现方式 | 逐文件搜索+读取 | 单次图查询 |
| Token 开销 | 读取整文件 | 精确符号+片段 |
| 结构理解 | Agent 自己推断 | 图谱蕴含结构 |
| 典型节省 | — | 47-99% token |

## 三种代码图谱的差异

本 wiki 已收录三种代码图谱模式：

| | 预索引知识图谱 | [[concepts/MCP代码智能\|MCP 代码智能]] | [[concepts/代码库图谱\|运行时 Trace 图谱]] |
|---|---|---|---|
| 代表工具 | [[entities/CodeGraph\|CodeGraph]] | [[entities/CodebaseMemoryMCP\|codebase-memory-mcp]] | [[entities/Hivemind\|Hivemind]] |
| 数据来源 | 静态 AST（tree-sitter） | Tree-sitter + LSP 语义 | Agent 运行时 trace |
| 更新方式 | 文件监控自动同步 | 按需索引 | trace 流实时 |
| 核心优势 | 快速、轻量、全覆盖 | 广度（158 语言）+ 深度（类型推断） | 反映真实使用模式 |
| 局限 | 静态分析无法捕获动态行为 | LSP 仅 10 种语言 | 依赖 agent 实际探索过的路径 |

三者互补而非互斥：CodeGraph 提供快速静态全貌，codebase-memory-mcp 提供深度语义，Hivemind 提供运行时真实连接。

## 关键设计决策

1. **预索引 vs 按需**：预索引在 Agent 启动前完成，查询时零延迟；按需索引在首次查询时构建，节省前置时间但增加首次查询延迟
2. **AST vs LSP**：AST 广度好、速度快；LSP 深度好、类型推断准确
3. **本地 vs 远程**：所有主流工具都选择 100% 本地（隐私、延迟、零成本）
4. **单工具 vs 多工具**：CodeGraph 倾向暴露单个全能工具 `codegraph_explore`，codebase-memory-mcp 暴露多个专用工具
