# 公司大脑（Company Brain）

**来源**：[[sources/yc-root-access-self-improving-company-2026-05-22]]
**提出者**：YC 合伙人（YC Root Access）

---

## 核心思想

> "All of your data, all of your emails, your DMs, the skills, the knowhow — that is the company brain. Humans sit around the edge of this, interfacing with the real world."

公司的核心资产不是软件，而是**中心化的知识体**：

```
            ┌───────────────────────────────┐
            │         公司大脑               │
            │  数据 + 邮件 + DMs            │
            │  + skills + knowhow           │
            │  + 录音 + 文档                │
            └──────────┬────────────────────┘
                       │
         人类围绕边缘，与现实世界接触
    ┌──────┴──────────────────────────┐
    │ 会议  销售  高风险决策  新颖情境  │
    └─────────────────────────────────┘
```

---

## 人类保留的场景

AI 进入大多数场景，但人类仍然不可替代于：
- **高风险、高情绪时刻**：联创关系破裂、重大战略决策
- **新颖情境**：没有先例的问题
- **伦理判断**：需要道德权衡的决策
- **面对面销售**：未来 20 年仍需人类在场

---

## YC 用户手册再生案例

Haj（YC 合伙人）实验：
- **输入**：3 个月积累的 2,000 小时 office hours 录音
- **流程**：提炼分类（融资/招聘/联创争议）→ 重新生成用户手册
- **输出**：150 页，质量显著优于原版（原版写于 5-10 年前）
- **自我改进**：每月更新，新建议与现有内容对比 → 采纳或丢弃
- **终态**：用户手册成为持续进化的活体大脑；接入 Agent → 获得 16 位 YC 合伙人的综合智慧

## 与 LLM Wiki 的同构

本 vault 就是一个公司大脑的实践：
- 所有摄入的源文件、wiki 页面 = 数据和 knowhow
- hot.md / index.md = 可查询接口
- wiki-ingest / wiki-query = 自我改进循环

## 在 AI 知识体系中的位置

公司大脑是 [[concepts/ProductOrganizationLayer边界]] 的组织知识组件：它把第 4 层 RAG、memory、skills 和 knowledge base 扩展到组织范围，使第 5 层 agent 能在真实业务上下文中行动。

稳定链路：`组织 artifact → 可查询知识 / memory / skills → agent workflow → trace / outcome → monthly or nightly consolidation → 更新公司大脑`。

没有第 6 层质量门，公司大脑会积累过时、冲突或未经授权的知识；没有第 7 层工作流，知识库又会变成静态文档，而不是产品和组织的运行底座。
