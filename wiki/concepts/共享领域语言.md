# 共享领域语言（CONTEXT.md）

Matt Pocock 提出的 AI agent 工作流模式：为 agent 建立项目的统一领域语言文档，解决"agent 太啰嗦/术语不一致"问题。

来源：Eric Evans 的领域驱动设计（DDD）"通用语言"概念，迁移到 AI 协作场景。

## 核心问题

Agent 被丢进项目后需要自己摸索术语，于是：
- 用 20 个词描述 1 个词能说清楚的事
- 变量、函数、文件命名不一致
- 每次会话都重新"学"同样的术语

## 解决方案：CONTEXT.md

一个描述项目领域语言的文档。

**示例**（Matt 的 course-video-manager 项目）：
- **Before**："There's a problem when a lesson inside a section of a course is made 'real' (i.e. given a spot in the file system)"
- **After**："There's a problem with the materialization cascade"

`CONTEXT.md` 通过 `/grill-with-docs`（[[concepts/Grilling模式]]的加强版）在与 agent 的对话中逐步构建，随 ADR（架构决策记录）一起维护。

## 三重收益

1. **命名一致性**：变量/函数/文件使用共享语言命名，codebase 内部一致
2. **导航效率**：agent 能更快找到相关代码，人类 review 也更容易
3. **Token 效率**：agent 思考时消耗更少 tokens（无需反复解释术语）

## 实践

```bash
# 通过 grill-with-docs 建立
/grill-with-docs
# agent 会问你关于当前任务的问题，同时更新 CONTEXT.md 和相关 ADR
```
