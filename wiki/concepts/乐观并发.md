# 乐观并发（Optimistic Concurrency）

**语境**：Anthropic Agent Memory API 中的多 Agent 并发写保护机制  
**来源**：[[sources/memory-dreaming-self-learning-agents-2026-05-11]]

---

## 原理

乐观并发假设大多数操作**不会冲突**，因此不在操作前加锁，而是在写入时验证：

1. Agent 读取 memory store，同时记录当前的 **content hash**（前置条件哈希）
2. Agent 完成本地计算，准备写回
3. 写入请求携带原始 content hash
4. 服务端检查：当前 hash == 请求携带的 hash？
   - **匹配** → 写入成功，更新 hash
   - **不匹配** → 写入拒绝（已被其他 Agent 修改），调用方需重新读取并重试

---

## 在 Agent Memory 中的意义

当数百/数千个 Agent 并发访问同一 memory store 时：
- **悲观锁**（全局锁）会造成严重排队等待，吞吐量极低
- **乐观并发**：允许并发读取，仅在冲突时拒绝写入，整体吞吐量高
- 冲突频率低（Agent 通常写入不同内容）→ 乐观假设成立
