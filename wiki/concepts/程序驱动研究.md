# 程序驱动研究（program.md 模式）

## 定义

用 **Markdown 文件（`program.md`）** 替代传统代码配置，作为研究员与 AI agent 之间的主要接口。人类用自然语言描述研究目标、约束和优先级；agent 读取后自主执行实验。

## 核心思想

> 研究员不再写 Python，而是写 `program.md`。

`program.md` 是 Karpathy 所说的"超轻量 skill"——它本身就是一个面向 agent 的指令集，而非面向 CPU 的代码。

## 接口设计

```
program.md
├── 研究目标（如：降低 val_bpb）
├── 探索方向（如：尝试不同注意力机制）
├── 约束条件（如：不改变 batch size）
├── 评估标准（如：val_bpb < 基线 0.05）
└── 特殊指令（如：优先尝试 Muon 优化器变体）
```

Agent 读取后，独立决定改什么、怎么改、何时回滚。

## 与传统配置的区别

| 维度 | 传统配置（YAML/JSON/argparse） | program.md |
|------|-------------------------------|------------|
| 受众 | 代码/解析器 | AI agent |
| 表达能力 | 结构化键值 | 自然语言 + 推理 |
| 灵活性 | 预先定义的参数空间 | 开放探索 |
| 人类可读 | 有限 | 完全自然语言 |
| 扩展性 | 需改代码 | 直接写新指令 |

## 在本 Vault 中的类比

本 wiki 的 `skills/*/SKILL.md` 与 `program.md` 模式同构：都是用 Markdown 给 agent 下指令，agent 读取后自主执行复杂任务。
