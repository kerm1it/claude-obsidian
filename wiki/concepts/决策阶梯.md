# 决策阶梯

**来源**：[[entities/Ponytail]]（Dietrich Gebert，2026）  
**所属体系层**：第 4 层（上下文与知识 / prompt engineering）

---

## 是什么

决策阶梯（Decision Ladder）是一种 Agent prompt-layer behavior constraint：Agent 在写任何代码之前，必须沿着 6 级阶梯逐级攀升，停在第一个适用的阶梯。目标是将"有经验的开发者不写不必要的代码"这一隐性直觉编码为显式 check-list。

---

## 六级阶梯

| 级 | 问题 | 行动 | 原则 |
|----|------|------|------|
| 1 | 这真需要存在吗？ | 跳过 | YAGNI |
| 2 | 标准库已有？ | 使用标准库 | stdlib first |
| 3 | 原生平台特性？ | 使用原生 | platform native |
| 4 | 已安装依赖覆盖？ | 使用已有依赖 | dep before new code |
| 5 | 一行代码就够？ | 写一行 | one-liner |
| 6 | 以上都不适用 | 写最小可行实现 | minimal |

---

## 不削减的底线

- 验证（validation）
- 错误处理（error handling）
- 安全（security）
- 可访问性（accessibility）

每个快捷方式用 `ponytail:` 注释标注升级路径，使技术债可见且可追踪。

---

## 与一般代码质量工具的区别

| 维度 | Linter / Dead Code Detector | 决策阶梯 |
|------|---------------------------|---------|
| 作用时机 | 代码写完后 | 代码写之前 |
| 机制 | 静态分析 | Prompt behavior constraint |
| 对象 | 已有代码 | Agent 行为 |
| 输出 | 警告/错误 | 阻止不必要的代码被生成 |

---

## 在 AI 知识体系中的位置

- **第 4 层（上下文与知识）**：决策阶梯是一种 prompt engineering 模式——将编码行为规范注入 Agent 的 system prompt / instruction
- **与 Skill 的关系**：Skill 告诉 Agent *怎么做*一类任务；决策阶梯告诉 Agent *先别急着写*。两者叠加：skill 提精度，ladder 减冗余
