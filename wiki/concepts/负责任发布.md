# 负责任发布

**来源**：Claude Mythos→Fable 5 发布范式（Anthropic，2026-06-12）  
**所属体系层**：第 7 层（产品与组织）

---

## 是什么

负责任发布是一种模型发布策略：当高能力模型在训练后评估中发现**过度危险能力**（如发现大量网络安全漏洞）时，不是直接发布模型，而是：

1. **评估**：完成训练和测试后评估模型的危险能力
2. **风险发现**：识别过度危险能力（excessively dangerous capabilities）
3. **内部修复**：将模型交给安全团队，在漏洞被利用前修复
4. **构建安全版**：基于该模型构建能力相当但 safeguards 更强的版本
5. **公开发布**：发布安全版本

---

## Mythos→Fable 5 的具体实例

```
Claude Mythos preview（训练完成）
  → 发现：能发现数千网络安全漏洞
  → 决策：不公开发布
  → 行动：交给保护关键软件的安全团队
  → 演化：基于 Mythos 构建 Fable 5
  → 增强：自动安全重定向（高风险→Opus 4.8）
  → 发布：Claude Fable 5
```

---

## 核心原则

1. **能力评估先于发布决策**：先确认模型能做什么（包括危险能力），再决定是否发布
2. **不因 fear 放弃能力**："it was the right call for the moment, but it was never the goal"——未发布是过渡状态，目标是安全发布
3. **分离能力与安全**：不削弱模型能力本身，通过 safeguards 控制风险
4. **渐进式释放**：先内部分发修复，再公开发布安全版
