# 设计概念（Design Concept）

来源：Frederick P. Brooks，《The Design of Design》

Matt Pocock 用此概念解释为何 AI 总做出你不想要的结果。

## 核心定义

当多个人共同设计某事时，他们之间漂浮着一个**无形的共同理念**——对正在构建的事物的"不可见理论"。这个共同理念不是一份资产、不能写进 markdown 文件，它是关于"我们在建什么"的无形共识。

这个无形共识叫做**设计概念（Design Concept）**。

## 在 AI 协作中的意义

你和 AI 之间没有共享的设计概念 → AI 的"需求采集"（requirements gathering）就是你们的对话过程 → 如果对话不够深入，AI 就会做出你想象中以外的东西。

Matt 的观察：Claude Code 的默认 Plan 模式急于生成计划文档（资产），但资产≠设计概念。设计概念是通过**充分的对话**建立起来的，而不是一个文件。

## 建立共同设计概念的方法

- [[concepts/Grilling模式]]（`/grill-me`）：让 agent 用 40-100 个问题"审讯"你，直到达到共同理解
- [[concepts/共享领域语言]]（CONTEXT.md）：用共同语言持久化对话成果
