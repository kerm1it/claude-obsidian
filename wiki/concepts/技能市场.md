# 技能市场

**来源**：[[entities/PMSkills]]（phuryn，2026）  
**所属体系层**：第 4 层（上下文与知识 / skill governance）

---

## 是什么

技能市场（Skill Marketplace）是 Agent Skills 的组织和分发模式：将特定领域的技能按**Skills → Commands → Plugins**三层打包，用户按需选装领域插件，而非一次性安装所有技能。

与技能仓库（Skill Repository）的区别：

| 维度 | 技能仓库 | 技能市场 |
|------|---------|---------|
| 安装模式 | 全量安装 | 按领域选装 |
| 组织方式 | 按功能分类 | 按领域打包 |
| 用户画像 | Agent 开发者 | 领域从业者（PM、研究者等） |
| 典型项目 | [[entities/ScientificAgentSkills]] | [[entities/PMSkills]] |
| 认知门槛 | 需了解所有 skills | 按需发现和安装 |

---

## 三层架构

```
Skill（原子技能，68 个）
  → 单一框架/方法的 Agent 工作流编码
  → 自动加载（上下文匹配）
  → 例：opportunity-solution-tree、porters-five-forces

Command（用户命令，42 个）
  → 串联多个 skills 完成端到端任务
  → 用户主动触发（/command）
  → 完成后建议关联命令
  → 例：/discover、/write-prd、/plan-launch

Plugin（可安装包，9 个）
  → 按领域分组 skills + commands
  → 用户按需安装
  → 例：pm-product-discovery、pm-execution
```

---

## 设计优势

1. **降低认知负载**：用户不需要知道 68 个 skills 分别是什么，只需知道"我需要做产品发现 → 装 pm-product-discovery"
2. **模块化治理**：每个 plugin 独立版本化、独立更新、独立安装
3. **领域适配**：不同垂直可以有不同的 plugin 结构——PM 按工作流阶段、科研按学科、工程按技术栈
4. **Commands 作为组合层**：skills 保持原子性，commands 表达"PM 实际怎么工作"

---

## 在 AI 知识体系中的位置

- **第 4 层（Skill Library）**：技能市场是 skill library routing 的分发端——定义了 skills 如何被打包、发现和安装
- **第 7 层（产品与组织）**：插件市场本身是产品形态设计——面向领域用户的界面、安装流程、更新机制
